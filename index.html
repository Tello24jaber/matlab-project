Phase 01 — The Premise

Engineering with Matrices

Problem Definition

Modern infrastructures—like corporate computer networks and urban road grids—contain thousands of interconnected nodes. Tracking paths or balancing loads manually is impossible.

However, these systems can be effectively abstracted into matrices and systems of linear equations.

Startup Objectives

  • Application 1 (Cybersecurity): Model malware spread paths through router connections using $k$-hop reachability graphs.
  • Application 2 (Urban Mobility): Solve traffic flow through city intersections using Gaussian elimination and free variables.
Phase 02 — The Mathematics

Bento Graph Theory

Before launching our products, let's establish the mathematical foundation.

1. Adjacency Matrices ($A$)

A network graph is mapped into an $n \times n$ matrix. If node $i$ connects to node $j$, $A_{i,j} = 1$. Otherwise, it is $0$.

2. Matrix Powers ($A^k$)

Multiplying a matrix by itself counts paths. The entry $(i,j)$ in $A^2$ gives the number of ways to travel from $i$ to $j$ in exactly 2 steps.

3. Conservation of Flow

For traffic geometry: $In = Out$. This physical law creates a linear system $Ax = u$. Linearly dependent rows yield free variables.

4. Cumulative Reachability ($H$)

To find all routers hit by a virus spreading $k$ times across the network, we sum the path matrices. If $H_{i,j} > 0$, router $j$ is computationally compromised.

H = A + A² + A³ + ... + Aᵏ

The Physics of Flow

$x_1 + x_2 = 250$
$x_1 - x_3 = 150$
$-x_2 + x_4 = 200$
$x_3 + x_4 = 300$

Because the rows are linearly dependent, we have infinite mathematical solutions. We isolate $x_2$ as the free variable.

x₁ = 250 - x₂ → 170
x₂ = 80 (Free)
x₃ = 100 - x₂ → 20
x₄ = 200 + x₂ → 280
Engineering Limit: Since $x_3 = 100 - x_2 \ge 0$, the maximum capacity for $x_2$ is 100.
Road Closure: If $x_2 = 0$, $x_1$ hits peak 250. The system absorbs the shock gracefully.
Phase 05 — Implementation

The Engine (MATLAB)

KHopReachability.m 
function [riskCounts, riskNodes, mostCritical, mostVulnerable, reachabilityMatrix] = KHopReachability(A, k)
    % Computes cumulative powers to find infection spread
    n = size(A,1);
    H = zeros(n, n);
    Apow = A;
    
    for p = 1:k
        if p > 1, Apow = Apow * A; end
        H = H + Apow;
    end
    
    % Convert physical path counts to True/False logic
    reachabilityMatrix = H > 0;
    
    % Remove self-risk
    for i = 1:n, reachabilityMatrix(i,i) = false; end
    
    % Find critical nodes via row/column summations
    [~, mostCritical] = max(sum(reachabilityMatrix, 2));
    [~, mostVulnerable] = max(sum(reachabilityMatrix, 1));
end
