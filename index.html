Phase 01 — The Premise

Engineering with Matrices

Problem Definition

Modern infrastructures—like corporate computer networks and urban road grids—contain thousands of interconnected nodes. Tracking paths or balancing loads manually is impossible.

However, these systems can be effectively abstracted into matrices and systems of linear equations.

Startup Objectives

  • Application 1 (Cybersecurity): Model malware spread paths through router connections using $k$-hop reachability graphs.
  • Application 2 (Urban Mobility): Solve traffic flow through city intersections using Gaussian elimination and free variables.
Phase 02 — The Mathematics

Bento Graph Theory

Before launching our products, let's establish the mathematical foundation.

1. Adjacency Matrices ($A$)

A network graph is mapped into an $n \times n$ matrix. If node $i$ connects to node $j$, $A_{i,j} = 1$. Otherwise, it is $0$.

2. Matrix Powers ($A^k$)

Multiplying a matrix by itself counts paths. The entry $(i,j)$ in $A^2$ gives the number of ways to travel from $i$ to $j$ in exactly 2 steps.

3. Conservation of Flow

For traffic geometry: $In = Out$. This physical law creates a linear system $Ax = u$. Linearly dependent rows yield free variables.

4. Cumulative Reachability ($H$)

To find all routers hit by a virus spreading $k$ times across the network, we sum the path matrices. If $H_{i,j} > 0$, router $j$ is computationally compromised.

H = A + A² + A³ + ... + Aᵏ

5. Gaussian Elimination

A system $Ax = b$ is solved by transforming the augmented matrix $[A|b]$ into Row Echelon Form (REF) using three elementary row operations:

  • $R_i \leftrightarrow R_j$ — swap two rows
  • $R_i \leftarrow c \cdot R_i$ — scale a row by $c \ne 0$
  • $R_i \leftarrow R_i + c \cdot R_j$ — add a multiple of one row to another

Back-substitution on the REF yields the solution. Further reduction to RREF gives a unique, readable form with leading 1s.

6. Rank & Free Variables

The rank of $A$ is the number of pivot columns after row reduction. If $\text{rank}(A) < n$ (number of unknowns), the system is under-determined:

# free variables = n − rank(A)

Free variables can take any real value. Each choice produces a valid solution — the solution space is a line, plane, or hyperplane (an affine subspace) rather than a single point.

In our traffic model: 4 unknowns, rank = 3 ⇒ exactly 1 free variable ($x_2$), yielding infinitely many physically valid traffic states.

7. Linear Independence & Dependence

A set of vectors $\{v_1, \dots, v_m\}$ is linearly dependent if at least one vector can be written as a linear combination of the others — equivalently, $c_1 v_1 + \cdots + c_m v_m = 0$ has a non-trivial solution.

In traffic flow, the four conservation equations are linearly dependent: summing all four gives $0 = 0$, meaning one equation is redundant. This is precisely why a free variable emerges and why the system has infinitely many solutions.

🧮 Mathematical Derivation — Malware Spread

Step 1 — Build the adjacency matrix $A$. Each row $i$ represents a server. $A_{i,j} = 1$ if server $i$ has a direct connection to server $j$, otherwise $0$. For $n = 10$ nodes this is a $10 \times 10$ matrix.

Step 2 — Compute matrix powers. The entry $\left(A^k\right)_{i,j}$ counts the number of distinct directed paths of length exactly $k$ from node $i$ to node $j$. This follows from the definition of matrix multiplication: each step selects an intermediate node.

Step 3 — Form the cumulative matrix. Define $H = A + A^2 + \cdots + A^k$. Then $H_{i,j} > 0$ if and only if there exists at least one directed path of length $\le k$ from $i$ to $j$. Converting $H$ to a boolean matrix gives the exact reachability set of every node within $k$ hops.

Step 4 — Identify critical nodes.

  • Most Critical Spreader = $\arg\max_i \sum_j H_{i,j}$ (node with the largest row sum — can reach the most others)
  • Most Vulnerable Node = $\arg\max_j \sum_i H_{i,j}$ (node with the largest column sum — reachable from the most sources)

Why matrix powers? A naive BFS traversal has complexity $O(n + e)$ per source. The matrix-power method amortises computation across all source nodes simultaneously: computing $A^k$ costs $O(n^3)$ but delivers the complete reachability map for every node at once — a critical advantage for large networks.

$H_{i,j} > 0 \Leftrightarrow \exists$  path $i \to j$  of length $\leq k$

Phase 05 — Implementation

The Engine (MATLAB)

KHopReachability.m 
function [riskCounts, riskNodes, mostCritical, mostVulnerable, reachabilityMatrix] = KHopReachability(A, k)
    % Computes cumulative powers to find infection spread
    n = size(A,1);
    H = zeros(n, n);
    Apow = A;
    
    for p = 1:k
        if p > 1, Apow = Apow * A; end
        H = H + Apow;
    end
    
    % Convert physical path counts to True/False logic
    reachabilityMatrix = H > 0;
    
    % Remove self-risk
    for i = 1:n, reachabilityMatrix(i,i) = false; end
    
    % Find critical nodes via row/column summations
    [~, mostCritical] = max(sum(reachabilityMatrix, 2));
    [~, mostVulnerable] = max(sum(reachabilityMatrix, 1));
end
Phase 06 — Results

Final Discussion

Two real systems. One algebraic backbone. Infinite applications.

01

Cybersecurity Network

By representing a 10-node network as an adjacency matrix $A$ and computing $H = \sum_{p=1}^{k} A^p$, we obtained the complete $k$-hop infection map in $O(n^3)$ time — far more efficient than per-node traversal. Row sums of $H$ identify the most dangerous spreader; column sums identify the most exposed node.

02

Urban Traffic Flow

Conservation of flow at each junction produced a $4 \times 4$ system with $\text{rank}(A) = 3$, confirming linear dependence. Gaussian elimination on $[A\,|\,\mathbf{b}]$ revealed exactly one free variable $x_2 \in [0, 100]$ — the full solution is a line segment of valid traffic states in $\mathbb{R}^4$.

◆ Key Insight

Both applications share the same algebraic backbone — encoding a network as a matrix transforms an intractable combinatorial problem into a tractable sequence of arithmetic operations that scale to any system size.
